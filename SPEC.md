# SPEC.md

Продуктова специфікація проєкту. Читати перед роботою над будь-якою фічею.

## Проєкт

Інтернет-магазин дитячих суконь преміум-сегменту. Україна, доставка Новою Поштою, оплата через Monobank Acquiring. Каталог 50–200 товарів. Більшість трафіку — мобільні (Instagram).

## Мови

- Українська — за замовчуванням, URL `/`
- Англійська — друга, URL `/en/...`
- hreflang для обох

## Структура сайту

- **Головна** — hero, добірка з каталогу, про бренд, CTA
- **Про нас** — бренд, шоурум, відгуки
- **Каталог: Сукні** — основна колекція, повний e-commerce
- **Каталог: Кутюрні сукні** — галерея + CTA "Отримати консультацію"
- **Каталог: Прикраси** — повний e-commerce
- **Умови замовлення** — доставка, оплата, повернення
- **Контакти** — телефон, месенджери, адреса + Google Maps, форма
- **Юридичні** — Публічна оферта, Політика конфіденційності

Соцмережі (Instagram, TikTok, YouTube) — у футері та контактах.
Плаваюча кнопка "Замовити дзвінок" на всіх сторінках → модалка → Telegram.

## Модель товару

```
Product {
  id, slug, category (dress | couture | accessory),
  name_uk, name_en, description_uk, description_en,
  images[],                  // Cloudinary
  colors[],                  // опції без впливу на ціну
  flags: { inStock, isNew, isBestseller, isHidden },
  variants[] {
    size,                    // діапазон у см, напр. "98-104"
    fabric,
    price,                   // різна для різних комбінацій
    images[]                 // опційно
  }
}
```

Кутюрні сукні — `category: couture`, без variants/price, тільки фото + CTA.

## Сторінка товару (dress / accessory)

- Галерея фото (mobile: swipe; desktop: thumbnails)
- Вибір: розмір → тканина → колір → прикраси (з каталогу accessory)
- Ціна оновлюється від комбінації
- Кнопки:
  - **Купити** → в кошик
  - **Уточнити терміни** → WhatsApp
  - **Пошив по своїм параметрам** → WhatsApp / Viber

## Сторінка кутюрної сукні

Тільки галерея фото + опис + кнопка **Отримати консультацію** → відкриває модалку для номера телефону → Telegram адміну.

## Кошик і checkout

- Guest checkout, без реєстрації клієнтів
- Кошик у localStorage (Zustand persist)
- Checkout-форма: ім'я, телефон, email, місто + відділення Нової Пошти (через NP API), примітка
- Оплата: Monobank Acquiring (redirect → callback з перевіркою підпису → підтвердження)
- Після успіху:
  - Сторінка подяки з деталями замовлення
  - Email клієнту через Resend
  - Telegram адміну з повними деталями замовлення

## Адмін-панель

Шлях `/admin`, Auth.js (email + пароль, один акаунт власника).

Розділи:
- **Товари** — CRUD, фото, варіації (size/fabric/price), прапорці (inStock, isNew, isBestseller, isHidden)
- **Категорії** — назви, slug, переклади
- **Сторінки** — Про нас, Контакти, Умови замовлення (редагування контенту)
- **Відгуки** — додати/редагувати/видалити (текст + ім'я + фото). Куратовані, без модерації користувацьких
- **Налаштування сайту** — телефон, посилання на месенджери та соцмережі, адреса шоуруму
- **Замовлення** — список з базовими статусами (new, paid, processing, shipped, done, cancelled)

## Інтеграції

| Сервіс | Призначення |
|---|---|
| Monobank Acquiring | Онлайн-оплата |
| Nova Poshta API | Вибір міста і відділення на checkout |
| Cloudinary | Зберігання та оптимізація фото |
| Telegram Bot (grammy) | Сповіщення про замовлення і заявки на дзвінок |
| Resend | Email клієнту (підтвердження замовлення) |
| Google Analytics 4 | Аналітика |
| Google Maps | Embed на сторінці контактів |

## Юридичне

- Cookie banner (Прийняти / Лише необхідні) → localStorage
- Тексти Публічної оферти та Політики конфіденційності — надає замовник

## Структура проєкту (орієнтовна)

```
/app
  /[locale]
    /(site)        — публічна частина
    /admin         — адмін-панель
  /api             — webhook (Mono), proxy (NP), internal
/components
  /ui              — shadcn primitives
  /features        — бізнес-компоненти
/lib               — утиліти, клієнти API
/prisma            — schema, migrations
/messages          — uk.json, en.json
/public
```
